# License files

**Select your Language: English | [简体中文](README.zh-CN.md)**

## Why this folder exists

This directory stores runtime license texts that are not otherwise available as
stable standalone files in the source tree.

The `licenses/` directory you see in a portable or installed copy of
Real-ESRGAN GUI is generated by `scripts/build-dist.ps1`. That generated folder
is the one users receive. It combines:

- the few license files maintained here,
- license files copied from the owning upstream source trees, and
- Microsoft runtime notices copied from the local build/runtime installation.

So this source folder is not meant to be a full copy of every final notice file.
It is the home for license texts that do not already have a better source path
inside the repository or build environment.

## Files maintained here

| Final file | Source reason |
| --- | --- |
| `Real-ESRGAN-LICENSE.txt` | Model files are extracted from the official Real-ESRGAN NCNN release archive, not from a checked-in Real-ESRGAN source tree. |
| `dirent-MIT-LICENSE.txt` | `third_party/ncnn_src/src/win32dirent.h` is compiled into the Windows backend and only carries a short header notice. |
| `stb-LICENSE.txt` | `third_party/ncnn_src/src/stb_image.h` and `stb_image_write.h` are compiled into the backend and carry embedded dual-license text rather than a standalone license file. |

## Files copied during release builds

These files are copied into the final runtime `licenses/` directory during the
release build. Keep the source path as the authority, so updates flow from the
component that owns the license.

| Final file | Build source |
| --- | --- |
| `Real-ESRGAN-ncnn-vulkan-Enhanced-LICENSE.txt` | `third_party/ncnn_src/LICENSE` |
| `ncnn-LICENSE.txt` | `third_party/ncnn_src/src/ncnn/LICENSE.txt` |
| `glslang-LICENSE.txt` | `third_party/ncnn_src/src/ncnn/glslang/LICENSE.txt` |
| `libwebp-COPYING.txt` | `third_party/ncnn_src/src/libwebp/COPYING` |
| `libwebp-PATENTS.txt` | `third_party/ncnn_src/src/libwebp/PATENTS` |
| `pybind11-LICENSE.txt` | `third_party/ncnn_src/src/ncnn/python/pybind11/LICENSE` |
| `Microsoft-dotnet-LICENSE.txt` | `LICENSE.txt` from the resolved .NET installation used for self-contained publish. |
| `Microsoft-dotnet-ThirdPartyNotices.txt` | `ThirdPartyNotices.txt` from the resolved .NET installation used for self-contained publish. |
| `Microsoft-Visual-Cpp-Redistributable-Redist.txt` | Visual Studio C++ Build Tools `Licenses/1033/Redist.txt`, matching the copied `engine/vcomp140.dll`. |

## Installer-only notices

Installer-only notice sources live under `packaging/windows/` because they are
shown by the installer rather than copied into the standard portable-folder
runtime. Optional Enigma boxed portable builds add
`licenses/Enigma-Virtual-Box-LICENSE.txt` only when Enigma is actually used.
